- **用途**：一句话说明
- **认证**：{需要 / 不需要 / 备注}

### 1.2 请求

#### Headers

| Name | 必填 | 说明 |
|------|------|------|
| `Content-Type` | 是 | `application/json` |

#### Path / Query

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|

#### Body

```json
{
  "field1": "value1",
  "field2": 123
}
```

| 字段 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| `field1` | string | 是 | - | 字段含义 |
| `field2` | integer | 否 | 0 | 取值范围 / 枚举 |

### 1.3 响应

#### 成功（200）

```json
{
  "code": 0,
  "data": { }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | integer | 0 表示成功 |
| `data` | object | 业务数据 |

#### 错误

| 错误码 | HTTP 状态 | 触发场景 |
|--------|-----------|---------|
| `XXX_NOT_FOUND` | 404 | 资源不存在 |
| `INVALID_PARAM` | 400 | 入参校验失败 |

### 1.4 示例

```http
POST /api/xxx HTTP/1.1
Content-Type: application/json

{
  "field1": "value"
}
```

```http
HTTP/1.1 200 OK
Content-Type: application/json

{
  "code": 0,
  "data": {}
}
```
