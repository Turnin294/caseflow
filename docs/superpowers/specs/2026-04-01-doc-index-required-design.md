# doc-index-required Skill 设计文档

- 日期：2026-04-01
- 作者：turnin
- 状态：已审批

---

## 背景与目标

随着项目文档持续增加，在没有预先了解现有文档结构的情况下直接编写文档，容易造成内容重复、边界模糊或体系混乱。

目标：在编写任何文档前，强制读取并分析文档索引，确保新内容与现有体系不冲突、不重叠。

---

## 适用范围

项目 `docs/` 目录下所有类型的文档，包括但不限于：
- 功能设计文档（`docs/design/`）
- Bug 分析记录（`docs/bug/`）
- API 接口文档（`docs/api/`）
- 其他自定义子目录文档

---

## 与现有 Skill 的分工

| Skill | 触发时机 | 职责 |
|-------|---------|------|
| `design-doc-required` | 写代码前 | 检查设计文档是否存在 |
| `doc-index-required` | 写文档前 | 读取索引，分析内容边界，避免重复 |

两者无交叉，各司其职。

---

## 执行流程

```
触发：准备写/创建 docs/ 下任何文档

1. 读索引
   ├─ 读 docs/INDEX.md（总索引，判断目标子目录）
   └─ 读 docs/{subdir}/INDEX.md（子目录详细索引）
        └─ 若不存在 → 引导初始化（提供模板，用户确认后写入）

2. 分析
   ├─ 目标内容是否已有对应文档？
   ├─ 是否与现有文档存在内容重叠？
   └─ 结论：新建文档 or 补充到已有文档
        └─ 有重叠 → 明确告知，建议合并或区分内容边界

3. 写文档
   └─ 按分析结论执行

4. 更新索引（半自动）
   ├─ 生成索引更新内容（新增条目 or 更新摘要/大纲）
   └─ 展示给用户确认后写入
```

---

## 索引文件格式

### `docs/INDEX.md`（总索引，轻量）

```markdown
# 文档索引

| 目录 | 说明 |
|------|------|
| [design/](design/INDEX.md) | 功能设计文档 |
| [bug/](bug/INDEX.md) | Bug 分析与修复记录 |
| [api/](api/INDEX.md) | API 接口文档 |
```

### `docs/{subdir}/INDEX.md`（子目录索引，含大纲）

```markdown
# {subdir} 文档索引

## {文档标题}
- 文件：`{文件名}.md`
- 摘要：{一句话描述文档内容}
- 大纲：{二级标题1} / {二级标题2} / {二级标题3} / ...
```

---

## 索引初始化模板

当子目录索引不存在时，向用户展示以下模板并引导确认写入：

```markdown
# {subdir} 文档索引

## {现有文档标题（自动扫描填入）}
- 文件：`{文件名}.md`
- 摘要：（请补充）
- 大纲：（请补充）
```

---

## 维护规则

- 新增文档后：在对应子目录索引追加条目，并更新总索引（如有新子目录）
- 修改文档标题/结构后：同步更新子目录索引中的摘要和大纲
- 删除文档后：从子目录索引移除对应条目
- 所有更新均为半自动：AI 生成建议内容，用户确认后写入

---

## 不适用场景

- 修改非 `docs/` 目录下的文件（如代码、配置）
- 对索引文件本身的更新（避免递归触发）
