# 开发日志 · 2026-04-03

## 修复 Skill · design-doc-required 未显式调用 doc-index-required（1.4.0 → 1.4.1）

**原因：** 发现 design-doc-required 整个正文中从未提到 doc-index-required，导致创建设计文档时索引未被触发。根因是：Claude 在执行 skill 流程时顺序跟着步骤走，不会因"即将写文件"而中途重新评估其他 skill 的 description。唯一可靠的方式是在 skill 正文中**显式点名调用**另一个 skill（bug-doc-required 已经这样做了，design-doc-required 漏掉了）。

**改动：**
- Mermaid 执行流程图：文档不存在分支加入 `调用 doc-index-required` 节点（创建前）和 `更新 INDEX.md` 节点（写完后）
- 第三步：明确要求"立即调用 doc-index-required"作为创建文档前的前置步骤
- 新增"第三·五步"：文档写完后调用 doc-index-required 第五步更新索引
- 新增"与其他 Skill 的协作关系"表（对齐 bug-doc-required 的结构）
- 红色警告表新增两行："直接建文件不查索引"、"doc-index-required 会自动触发"

**影响：**
- design-doc-required 执行时将强制在建文件前调用 doc-index-required 读索引、写完后更新索引
- 与 bug-doc-required 对齐，两者现在都有完整的 skill 协作链

## 修改 Skill · 触发时机全面重构（参照 superpowers 插件模式）

**原因：** 发现多个 Skill 因 description 描述触发时机过晚，导致 Claude 在用户提出需求/分析请求时未自动触发。根因是 description 都在说 "about to write code"，但需求讨论和可行性分析阶段早于写代码，Claude 判断不需要触发。参照 superpowers 插件的 brainstorming skill，后者在 description 里描述的是更早的意图阶段，并在正文中加入 `<HARD-GATE>` 硬锁。

**改动：**

### skills/design-doc-required/SKILL.md
- description：从"about to write code"改为覆盖"提出需求/分析可行性/讨论架构"等更早阶段，列举具体触发短语
- 正文：在核心原则后加入 `<HARD-GATE>` 块，明确禁止在文档检查完成前进行任何分析或方案讨论

### skills/bug-doc-required/SKILL.md
- description：从"writing a bug analysis document"改为覆盖"用户报告 bug/描述异常/请求分析"等更早阶段，列举具体触发短语

### skills/doc-index-required/SKILL.md
- description：加强为"BEFORE creating ANY file under docs/ — no exceptions"，明确指出其他 Skill 产出 docs/ 文件时也必须先触发本 Skill

### skills/pre-implementation-code-orientation/SKILL.md
- description：明确定位为"design-doc-required 或 bug-doc-required 完成后、第一行实现代码之前"的桥接步骤，列举触发短语

### CLAUDE.md
- 新增「Skill 主动触发规范」章节，列出各意图场景与对应 Skill 的映射表，强调"触发时机是用户表达意图的那一刻，而不是开始动手的那一刻"

### .claude-plugin/plugin.json + marketplace.json
- version: 1.3.0 → 1.4.0

**影响：**
- Claude 将在用户提出需求的第一时间触发 design-doc-required，而非等到写代码时
- Claude 将在用户报告 bug 的第一时间触发 bug-doc-required
- doc-index-required 将在所有 docs/ 文件操作前被触发，包括其他 Skill 内部产出的文件
