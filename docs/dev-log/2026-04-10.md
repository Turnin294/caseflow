# 开发日志 · 2026-04-10

## 16:00 · 新增 Skill · business-logic-orientation

**原因：** 在重构退款退货模块时发现，重构前缺少对现有业务逻辑的系统化梳理规范。开发需要快速回顾逻辑、产品需要了解业务流程、AI 需要快速检索代码坐标，三类读者的需求各不相同。现有 skill（design-doc-required 面向"要改成什么"，bug-doc-required 面向"出了什么问题"）均不覆盖"现状是什么"这个维度。

**改动：**
- 新建 `skills/business-logic-orientation/SKILL.md` — 主规范文件，定义执行流程、标准章节结构（7 通用章节 + 5 后端附加章节）、AI 速查索引规范、代码阅读规范、红色警告
- 新建 `skills/business-logic-orientation/template.md` — 完整梳理文档模板，含场景总览矩阵、核心代码索引、场景 3 图（时序图/流程图/泳道图）、知识图谱、业务规则速查表、后端附加表操作矩阵和状态扭转明细
- 新建 `skills/business-logic-orientation/ai-ref-template.md` — AI 速查索引模板，7 章节紧凑结构（场景决策树、文件索引、方法索引、调用链索引、表操作索引、状态扭转索引、业务规则索引）
- 更新 `CLAUDE.md` — Skill 索引表新增一行、辅助资源表新增两行、主动触发规范表新增一行
- 更新 `README.md` — Skills 表新增一行
- 更新 `docs/skill-flow.md` — v4 -> v5：Skill 总览表新增一行、调用关系图新增重构入口路径和 BLO 节点、常见困惑速查新增两条
- 创建 `docs/skill-flow-20260404-v4.md` — v4 版本快照
- 更新 `.claude-plugin/plugin.json` — version: 1.4.2 -> 1.5.0
- 更新 `.claude-plugin/marketplace.json` — version: 1.4.2 -> 1.5.0

**影响：**
- 今后用户表达重构/复写/迁移意图时，自动触发此 Skill
- 产出两份文档：完整梳理文档（人类阅读）+ AI 速查索引（AI 快速检索，节省 70%+ token）
- 与 design-doc-required 形成上下游关系：先 business-logic-orientation 梳理现状，再 design-doc-required 规划重构方案
- 后端项目额外要求表操作矩阵和状态扭转明细，确保数据库变更回归可验证

## 21:00 · 修复 · design-doc-required / pre-implementation-code-orientation 触发短语盲区

**原因：** 用户反馈"阅读文档帮我修改代码"时 Claude 跳过了设计文档检查直接改代码。排查发现两个 skill 的 description 触发短语只覆盖了"新需求/重构计划/方案讨论"等场景，没有覆盖"帮我改代码""根据文档改代码"等直接编码请求。Claude 匹配不到触发短语就跳过了 skill。

**改动：**
- 修改 `skills/design-doc-required/SKILL.md` description — 增加代码修改类触发短语（中英文）、增加兜底规则（即将 Edit/Write 源码文件时若未触发本 skill 则必须先触发）、红色警告表新增 3 条
- 修改 `skills/pre-implementation-code-orientation/SKILL.md` description — 增加中英文代码修改类触发短语
- 修改 `CLAUDE.md` — 主动触发规范表新增「请求修改/编写代码」行、新增「兜底规则」段落
- 更新 `docs/skill-flow.md` — design-doc-required 和 pre-implementation-code-orientation 触发时机描述增强
- 更新 `.claude-plugin/plugin.json` — version: 1.5.0 -> 1.5.1
- 更新 `.claude-plugin/marketplace.json` — version: 1.5.0 -> 1.5.1

**影响：**
- 今后用户以任何形式表达"帮我改代码"意图时，Claude 都会先触发 design-doc-required 进行设计文档检查
- 即使触发短语未匹配，CLAUDE.md 的兜底规则也会在 Claude 即将执行源码 Edit/Write 前强制拦截
