## 11:10 · 修改 Skill · coding-standards-common §7「知识 vs 逻辑」DRY 二分

**原因：** 上述反模式里「同一映射约定抄三处」本质是 DRY 被误读：DRY 的本意(《程序员修炼之道》)是**知识**唯一权威表示，不是"代码不能重复"。旧 §7「DRY but rule of 3」只覆盖"逻辑重复"那半(且偏向"别过早抽象"),缺了"知识/约定/常量第一次就该单源"那半——正是这次踩坑的方向。另补一条通用纪律：不熟悉的外部 API 先核验签名再写(本次写过不存在的 import)。

**改动：**
- §7 重写为「先分知识还是逻辑」：知识/约定/常量/契约 → SSOT 第一次就单源；逻辑/代码结构 → rule of 3。
- 新增 §7.5「外部 API / 不熟悉的库先核验不臆造」。
- 自检清单同步两条。

**影响：**
- 这是语言无关的通用准则，对所有源码生效；与新 skill 的「约定 SSOT」「枚举输出」是母原则与具体落点的关系。
