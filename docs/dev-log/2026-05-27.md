# 开发日志 · 2026-05-27

## 11:30 · 调整流程 · init-project-docs 输出根反转到用户目录知识库

**原因：** init-project-docs 是唯一还把知识图谱直接生成进项目 `docs/` 的产出方，与 design-doc-required / bug-doc-required / business-logic-orientation / daily-work-log 自 v1.20 起默认写 `{USER_DOCUMENTS}/ai-docs/{project}/` 的约定不一致，会污染源项目仓库。用户决策：知识图谱只用 ai-docs、彻底不碰 docs/——知识图谱不再随项目仓库分发，避免污染源项目。

**改动：**
- init-project-docs/SKILL.md 新增「输出根目录」节，定义 `{KG_ROOT} = {USER_DOCUMENTS}/ai-docs/{project}/`；Phase 1-4 所有产出表、前置条件、存储结构树的 `docs/` 前缀全部改为 `{KG_ROOT}/`
- templates/00_project_overview.md 文档导航、templates/02_module_map.md 详细文档链接改为相对文件名（去掉 `docs/` 前缀），使整套文档与所在根解耦
- 读取方同步改读用户目录：design-doc-required Step 0、bug-doc-required Step 0 的 `docs/00_project_overview.md` → `{USER_DOCUMENTS}/ai-docs/{project}/00_project_overview.md`
- project-docs-update 前置条件 + 流程图 + 5 个检测项对比路径全部改 `{KG_ROOT}/`
- CLAUDE.md / AGENTS.md init-project-docs 索引行补充输出路径说明与关键词；skill-flow.md Step 0 FAQ 同步

**影响：**
- 今后 init-project-docs 生成的全部知识图谱文档（00-10 numbered、modules/、skills/、development-reference）默认落用户目录，源项目 `docs/` 不再被写入
- design-doc/bug-doc 的 Step 0 知识图谱预热改从用户目录读取；项目无知识图谱时仍自动跳过、向后兼容
- 历史已在 `docs/` 生成知识图谱的项目需用户自行迁移到 ai-docs；本 skill 默认只读写 ai-docs
- 不涉及 backend-knowledge-graph / reverse-index / glossary 的 `docs/knowledge-graph/` 正式版（其 candidate 池本就在 ai-docs，正式版去留是另一决策，本次不动）
