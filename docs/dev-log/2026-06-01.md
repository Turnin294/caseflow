## 新增 `dart-coding-standards` skill：Dart 注释规范与 Java 对称，闭合「Dart 注释按指定语言来」缺口

**原因:** 用户要求 plugin「完美跑通：Java / Dart 注释按指定语言来，非这两个语言按通用标准」。深度审视（3 个 Explore agent 并行审计 + 人工核实）发现两个问题：

1. **不对称缺口**：Java 有 `java-coding-standards`（Javadoc 独占条款），但 Dart 的 dartdoc 专属指引此前**只存在于 `korepos-backend-service`**，而它的触发范围仅限 `lib/features/{module}/backend/`。普通 Flutter / UI 代码（presentation 等）只吃到通用 `coding-standards-common §5` 的一句「有 doc comment 语法优先用」，没有 dartdoc 专属指引——和 Java 不对称，「Dart 注释按指定语言来」只对 backend 成立。
2. **单一来源被破坏（硬矛盾）**：`korepos-backend-service` SKILL.md:49 与 README.md:204 仍要求 DTO 新增字段标 `[ADDED YYYY-MM-DD vN]`，直接违反 v1.28.8 立的 §5.4 红线（明禁 `[ADDED]` + 日期标记）。这是 v1.28.8 单一来源化时漏改的存量。

**改动:**
- **新建 `skills/dart-coding-standards/SKILL.md`**（与 java-coding-standards 对称定位）：只写 Dart 独占的 Effective Dart / dartdoc 条款——文档注释用 `///`（不用 `//`）、首句独立摘要、引用标识符用 `[]`、写在注解之上、**不用 Javadoc 的 `@param`/`@return`/`@throws`**（Dart 用散文 + `[参数名]`）、私有成员（`_` 前缀）dartdoc 从简禁堆契约史、Flutter Widget 注释讲「展示什么 / 何时变化」、TODO `TODO(负责人): 原因` 禁日期、金额禁 `double`。通用 7 条 + 注释三档 + 红线全部 delegate 到 `coding-standards-common`，不重复。
- **修单一来源矛盾**：删除 korepos-backend-service SKILL.md / README.md 里的 `[ADDED YYYY-MM-DD vN]` 字段标注要求，改为「禁带 `[ADDED]` / 日期 / 版本标记，变更历史归 git / design doc，见 §5.4」。
- **语言路由三层定型**：`coding-standards-common`（通用，所有语言）→ `dart-coding-standards`（所有 .dart 的 dartdoc 语法）→ `korepos-backend-service`（仅 korepos backend 接口的 DTO/service/DAO 专属约束）。CLAUDE.md 触发表、分类导航 ③、核心调用顺序第 12 步、冲突解决（dart vs common / dart vs korepos）、Skill 索引、skill 计数 25→26 全部同步；README 与 docs/skill-flow.md 同步（含修正「新写 Dart 只走 common」这条已过期 FAQ）。

**影响:**
- 「Java / Dart 注释按指定语言来，其余按通用标准」三层路由首次完整成立：TS/Python/Go/Kotlin 仍只走 common §5；Java 走 common + java；Dart 走 common + dart（korepos backend 再叠加 korepos-backend-service）。
- 单一来源完整性恢复：korepos-backend-service 不再与 §5.4 红线冲突；注释红线仍唯一收口在 coding-standards-common §5.4 / §5.4.1。
- 触发链路不变（common 先于具体语言 skill），只是 Dart 多了一层专属叠加。
