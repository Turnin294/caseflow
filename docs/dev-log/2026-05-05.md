## 新增 Skill · coding-standards-common

**原因：** 项目里实际写代码并不只有 Java（前端 TS / Vue、Flutter Dart、Python 脚本等都在写），但插件之前只有一份 `java-coding-standards`，描述里写的是"Use when writing Java code"，所以改 ts/dart 时根本不触发，导致跨语言代码缺乏一致的命名 / 函数原子 / 层次 / 零魔法值 / 注释 / 异常铁律。同时，原 Java skill 内 §1 命名 / §2 格式 / §3 注释 / §7 异常的大半条款其实都是跨语言通用的，把它们留在 Java skill 里既造成了"非 Java 不约束"的盲区，又让团队为别的语言新建 skill 时只能复制粘贴。

**改动：**
- 新增 `skills/coding-standards-common/SKILL.md`，作为跨语言通用底，囊括 7 条铁律：
  1. 命名表意（禁拼音 / 接口实现 `XxxImpl` / `XxxDO/DTO/VO` 不混 / `get/list/count/save/remove/update` 前缀规范）
  2. 函数原子（80 行硬阈值 + ≤4 参数 + ≤3 嵌套，超出拆 `_xxxStep` 私有方法）
  3. 层次分明（上层调下层、UI/Controller 禁直 SQL/HTTP、同层禁互调、跨 feature 沉原子能力层）
  4. 零魔法值（DB 字段值 / 协议码 / 状态机数字强制枚举，例外仅 `0/1/-1`、`true/false`、空串、单测字面量）
  5. **注释三档铁律**（类 1-3 行 + 方法 1-2 行 + 核心代码块 1 行；禁注释代码 / 禁变更日志 / 禁段落式设计史 / 禁重复函数名废话；TODO 必带原因负责人）
  6. 异常不静默（catch 不空吞、日志带堆栈、finally 禁 return、有事务 catch 须回滚）
  7. 删冗余 / DRY but rule of 3（三处才抽公共、dead code 直接删、不预设未来扩展）
- `skills/java-coding-standards/SKILL.md` 同步瘦身：删除与 common 重复的 §1 命名通用部分 / §2 格式 / §3 注释通用部分 / §7 异常通用部分，仅保留 Java 独占条款（Javadoc 语法、POJO 布尔 is 前缀、Integer == 比较、BigDecimal、StringBuilder 循环、@Override、`Arrays.asList`、`HashMap` 容量、`Map.entrySet`、线程池、`SimpleDateFormat` static 禁止、`ThreadLocal.remove`、SLF4J 占位符、关系库 SQL 规范、PreparedStatement）。
- CLAUDE.md / AGENTS.md：主动触发表新增"开始编写或修改任何源码前 → coding-standards-common（先于具体语言 skill）"；Skill 索引表新增 `coding-standards-common` 行，更新 `java-coding-standards` 行描述为"Java 独占条款，通用 7 条 delegate 到 coding-standards-common"。
- README.md：能力清单首条加"跨语言通用编码规范"；Skills 表新增 `coding-standards-common` 行。
- docs/skill-flow.md：Skill 总览新增 `coding-standards-common` 行；调用关系图、功能开发完整链路、Bug 修复完整链路三张 mermaid 图都在 `architecture-ddd-lite-fullstack` 之后、`java-coding-standards` 之前插入 `CSC` 节点；FAQ 新增"common 和 java-coding 区别"+"注释到底要写多少"两条。创建 `docs/skill-flow-20260505-v20.md` 快照。

**注释立场综合的依据：** 团队历史规范来自阿里黄山版（每个类每个方法都要 Javadoc，强调"全员都要写"），而 `bugfix-coding-style` v1.17 已经反转为"禁源码内变更日志、函数头不堆复盘、复杂逻辑就近 WHY"。两者表面冲突，但实际可以折中：
- 保留阿里方向的"**全员都要写**"（类 / 方法 / 核心块都必须有注释），照顾"代码不易读取"的实际诉求
- 吸收 Clean Code 方向的"**写 WHY、要短**"（类 1-3 行、方法 1-2 行、核心块 1 行；禁堆段落 / 禁变更日志 / 禁实现步骤流水）
- 简要原则兜底：写不下就说明你想塞实现细节，那部分应该进 design doc 而不是源码

如此 `coding-standards-common §5` 与 `bugfix-coding-style` 的方向完全对齐，不再相互冲突。

**版本：**
- `.claude-plugin/plugin.json` + `marketplace.json`: 1.20.0 → 1.21.0（新增 Skill = Minor）
- `.codex-plugin/plugin.json`: 1.12.0 → 1.13.0

**影响：**
- 后续写 TS / Dart / Python / Vue / React 等代码时也会自动应用 7 条铁律（之前只有 Java 走规范）
- 新建语言 skill（如 typescript / dart / python coding standards）只需写**独占条款**，通用部分由 common 承担
- Java skill 文件从 ~180 行降到 ~140 行，更聚焦
