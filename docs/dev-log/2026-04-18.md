## 17:20 · 修改 Skill · markdown-writing-standards

**原因：** 用户在整理 korepos 项目的《环境搭建与常见问题》文档时发现：原文档 13 个平坦 `##` 章节里混杂了「环境搭建 / 日常开发 / 报错速查 / 已过期附录」四类不同性质内容，不同身份的读者（新人、老手、踩坑排查）不知道自己该看哪段。用户把文档拆成 Part I/II/III 三段后反馈："我有没有那个 skill 规约就是规定你编写 markdown 的时候要过一下 markdown 目录，进行一次分析整合目录，避免重复目录、目录散乱、没有分类的情况"—— 现有 `markdown-writing-standards` 只覆盖到 Mermaid 语法 + 表格 + 代码块 + 标题层级（第 6 节只有两小节：标题层级、章节之间），**完全没有"结构性改动后过一遍 TOC"这条规则**，属于规约空白。本次补齐。

**改动：**
- `skills/markdown-writing-standards/SKILL.md`：
  - frontmatter `description` 增加 TOC Review 触发语义（`ALSO TRIGGER after completing any structural edit ... to perform TOC review`）
  - 第 6 节新增 `6.3 目录结构复核（TOC Review）` 子章节，含 `<HARD-GATE>`、触发条件、三步复核流程（列 TOC 快照 → 8 项排查表 → 判断是否需要快速导航）、复核自检清单
  - 8 项排查表覆盖：分类混杂 / 内容重复 / 标题重复 / 层级断层 / 编号错乱 / 命名风格不统一 / 单章节超长 / 交叉引用失效 / 超过 4 级嵌套
- `CLAUDE.md`：
  - Skill 主动触发表的 markdown-writing-standards 一行，触发条件从"生成或修改包含 Mermaid 图表的 Markdown 内容"扩展为"...；或完成 Markdown 文件的结构性写入/重组（新增/删除/重命名 ##、### 章节，或章节移动/合并）"
  - Skill 索引表的覆盖范围追加"目录结构复核（TOC Review — 分类混杂/重复/层级断层/交叉引用失效/快速导航判断）"，关键词追加"目录结构、TOC、章节重构、目录复核"
- `docs/skill-flow.md`：
  - Skill 总览表 markdown-writing-standards 触发时机一行，补充"完成 Markdown 文件的结构性写入/重组后做目录复核"
  - 调用关系图里 `MWS` 节点 label 从 "Mermaid 语法自检" 扩展为 "Mermaid 语法自检 + 目录复核"

**影响：**
- 今后 AI 完成任何 Markdown 文件的章节结构性编辑（新增/删除/重命名 `##`/`###`、章节移动合并）后，必须自动触发一次 TOC 复核，按 6.3.2 的 8 项排查表过一遍，再用 6.3.3 自检清单确认
- 变更属于轻微（新增子章节，未改动 skill 链路结构），未创建 skill-flow 版本快照
- 发版时需 Patch 位递增（按 caseflow/CLAUDE.md 发版规则），push 前还需同步 `.claude-plugin/plugin.json` 和 `marketplace.json` 的 version 字段、以及 README.md「包含的 Skills」表如有必要

## 19:30 · 修改 Skill · bug-doc-required

**原因：** 用户在 korepos-refund 项目编写《订单附加费必填字段缺失》bug 文档时指出两个规约空白：

1. 现有规则要求用「英文 kebab-case」命名 bug 目录/文档，但项目实际 `docs/design/` 下模块全部用中文命名（如「退款退货逻辑重构」「反结账」），bug 目录的英文命名与 design 的中文命名风格割裂，跳来跳去不顺手。
2. `docs/bug/` 下现为「一级扁平结构」：`docs/bug/{bug名}/{bug名}.md`，缺少按业务模块分组的二级目录。同一模块（比如退款退货重构）后续会累积多条 bug，扁平堆放缺乏归类。

用户原话："bug的内容所有内容都是英文命名 改为使用中文 然后如果是对应design下某个模块的bug对应bug下也建立一样的目录 记录模块下的各类BUG BUG文档名称也是中文定义"。

**改动：**

- `skills/bug-doc-required/SKILL.md`：
  - **目录结构章节** 重写：从一级扁平改为三级结构 `docs/bug/{模块名}/{bug名称}/{bug名称}.md`；模块名必须与 `docs/design/{模块名}/` 完全一致（对齐 design 的中文命名）；无对应 design 模块时退化为一级扁平作为兜底；新增模块级 `INDEX.md` 子索引要求。
  - **命名规则** 改为强制中文：由英文 kebab-case 改为中文命名，与 design 模块命名风格保持一致；历史遗留英文目录不强制迁移。
  - **执行流程图** 新增「扫描 `docs/design/` 判断是否有对应模块」分支节点，驱动归档到模块目录或一级扁平。
  - **红色警告表** 新增三条：中文命名强制、扫描 design 强制、模块名不得同义替换。
- `CLAUDE.md` Skill 索引表 bug-doc-required 行：覆盖范围追加「目录按模块分组（对齐 `docs/design/{模块名}/`，三级结构）」和「目录与文件名使用中文命名」；关键词追加「模块分组、中文命名」。
- `README.md` 「包含的 Skills」表 bug-doc-required 行：作用列追加「目录按模块分组（对齐 `docs/design/`）；中文命名」。
- `docs/skill-flow.md` 头部：`最后更新` 升版 v6.1 → v6.2，变更摘要改写为「目录结构三级化 + 中文命名」，上一版摘要顺延。流程节点无变化，未创建版本快照。
- `.claude-plugin/plugin.json` 与 `.claude-plugin/marketplace.json`：version `1.8.1` → `1.8.2`（Patch，规则细化）。

**影响：**

- 今后在任何项目下（尤其 korepos 系）新建 bug 文档时，必须先扫描 `docs/design/`：有对应模块 → `docs/bug/{模块名}/{bug中文名}/{bug中文名}.md`；没有 → `docs/bug/{bug中文名}/{bug中文名}.md`。
- korepos-refund 当天刚写的 `docs/bug/order-additional-fee-missing-required-fields/` 属于新规则前命名，暂未迁移（规则明确：历史遗留不强制迁移，但用户本次同时会批准按新规则重命名为 `docs/bug/退款退货逻辑重构/订单附加费必填字段缺失/订单附加费必填字段缺失.md`）。
- 流程节点未变，属轻微变更，未新建 `skill-flow-{YYYYMMDD}-v{N}.md` 快照。
