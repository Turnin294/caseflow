**原因：** 实践中遇到一类未被现有规则覆盖的改动 —— 设计文档已存在、改动属于该文档覆盖范围内的修正/对齐/删冗余（如 selectedServices 强制覆盖 bug，与云端 calculateRefundPrice 不一致）。当前 SKILL 只有两档：要么走"新建 vN+1 + coding.md"重型流程，要么走"完全跳过文档检查"的合法例外，中间档缺位。直接走完整流程对小修来说成本过高，跳过又会让设计文档与代码描述失真。

- 同步更新执行流程图，新增 LIGHT 判定节点
- 改写「合法的例外情况」章节，明确"轻量修订流水 / 新建版本 / 完全跳过"三档边界

**影响：**

## 18:20 · 新增 Skill · bugfix-coding-style

**原因：** 上一条 selectedServices 修复时，第一版直接 `Edit` 删掉 6 行 if 块，被用户拦下要求"先注释原错误代码，追加新代码，联调期保留"。这个规则不限于本次 bug：联调期所有 bug 修复都需要保留旧代码作为回滚锚点 + code review 对照参照 + 跨端联调讨论凭据。现有 `java-coding-standards`（仅 Java）、`pre-implementation-code-orientation`（读文档坐标）、`coding-violation-log`（反应式登记）都不适合塞这条主动规则，按 CLAUDE.md 决策流程归为"完全独立的规范领域"，新建独立 skill。

**改动：**

- 新建 `skills/bugfix-coding-style/SKILL.md`：核心规则 + 适用范围矩阵 + 注释格式三要素（`[DEPRECATED YYYY-MM-DD]` + 替代摘要 + 引用文档）+ 各语言注释符 + 摆放位置（新代码在前，注释在后）+ 移除时机（仅由用户明确指令）+ 红色警告
- 更新 `CLAUDE.md` 主动触发表新增一行（联调期 bug 修复触发 bugfix-coding-style）
- 更新 `CLAUDE.md` Skill 索引表新增一行（覆盖范围 + 关键词）
- 在 `design-doc-required` 第四·五步「处理动作」加第 5 条：进入代码改动必须遵循 bugfix-coding-style
- 创建 `docs/skill-flow-20260425-v8.md` 快照（v7 → v8）
- 更新 `docs/skill-flow.md`：头部 blockquote 标记 v8、Skill 总览表新增一行、Bug 修复链路图新增轻量修订流水分支 + bugfix-coding-style 节点、常见困惑速查新增 3 条 Q&A
- 修复用例代码风格：`korepos-refund/lib/features/refund/backendv2/service/refund_price_service.dart` 把直接删除的 if 块改回"注释保留 + 追加新代码"形式

**影响：**

## 18:55 · 修改 Skill · bugfix-coding-style（v1 → v1.1，新增 B 类规范）

**原因：** 联调期 selectedTip 缺漏修复时暴露原 skill 盲区 —— skill 只覆盖了"修改/替换/删除原错误代码"（A 类），但实际联调里大量改动是"补上原本就缺漏的对齐代码"（B 类，无原代码可注释，比如云端有但本地没搬过来的后处理逻辑）。B 类如果不加任何标记直接 push，reviewer 看到纯新增代码无法判断"以前为什么没有、现在为什么必须加"，commit 信息也讲不清。

**改动：**

- 在 `skills/bugfix-coding-style/SKILL.md` 顶部"核心原则"加 A/B 两类总表，明确分别有强制注释规范
- description 字段同步扩到覆盖 B 类触发词（`补缺漏`、`补上漏掉的逻辑`、`add missing piece`）
- "适用范围"表新增列「改动类型」，标注每个场景属 A / B / A+B
- "注释格式规范"章节拆为 A 类 / B 类两节，B 类标准模板用本次 selectedTip 修复实例
- 新增「A vs B 判定速查」对照表（4 项判定维度）
- "移除时机"表扩成 A/B 两列，明确 B 类清理只删头注释保留代码
- "红色警告"表新增两条："纯新增不需要注释"（错） + "B 类清理时把代码也删了"（错）
- 同步更新 `CLAUDE.md` Skill 索引表对应行，覆盖范围补 B 类规则；关键词加"补缺漏 / ADDED"

**影响：**

- 联调期"补上缺漏对齐代码"明确归入本 skill 治理，不再需要新建 skill
- 第二个用例：`refund_price_service.dart:368-380` selectedTip 后处理，使用 B 类 `[ADDED 2026-04-25]` 头注释规范
- v6 调整流水追加第二条（小费 refundAmount 缺漏修正）
- 不影响 A 类既有逻辑；A 类用例仍指向 selectedServices 那次修复
